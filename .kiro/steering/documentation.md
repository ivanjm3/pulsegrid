# Documentation Steering

Whenever you create, modify, or delete code:

1. Check whether the documentation is affected.
2. If no documentation available create it
2. If yes, update the relevant documentation files.

Documentation includes:
- README.md
- docs/architecture.md
- docs/api.md
- docs/flow.md
- docs/interview-notes.md

Always keep documentation synchronized with the implementation.

Never leave documentation stale.

For every significant feature include:

- Purpose
- Architecture
- Request flow
- Important structs/classes
- Functions added
- Configuration changes
- Environment variables
- Design decisions
